Примечание: измерение производилось на массивах до 50000 элементов (дальше ноутбук не потянул к сожалению)

Общие выводы по производительности алгоритмов:
1. Влияние типа входных данных:
•	Random Arrays: Все алгоритмы показывают стабильное O(n log n) поведение
•	AlmostSorted Arrays: Гибридные версии демонстрируют значительное ускорение благодаря эффективности Insertion Sort на частично упорядоченных данных
•	ReverseSorted Arrays: Стандартный Merge Sort показывает худшую производительность, в то время как гибридные версии работают лучше
2. Сравнение алгоритмов:
Standard Merge Sort:
Стабильная производительность на всех типах данных
•	Наименее эффективен на частично упорядоченных массивах
•	Высокие накладные расходы на рекурсию для маленьких подмассивов
Hybrid Merge Sort (Threshold=10):
• Лучшая производительность на AlmostSorted массивах
•	Может быть менее эффективен на полностью случайных данных из-за частых переключений
Hybrid Merge Sort (Threshold=50):
•	Оптимальный баланс для большинства сценариев
•	Значительное ускорение на частично упорядоченных данных
•	Снижение накладных расходов рекурсии
3. Влияние порогового значения (Threshold):
Threshold=10:
•	Больше переключений на Insertion Sort
•	Лучше для почти отсортированных массивов
•	Хуже для полностью случайных данных (лишние переключения)
Threshold=50:
•	Оптимальное значение
•	Достаточно большой для эффективности Insertion Sort
•	Достаточно маленький для минимизации накладных расходов
4. Количественные улучшения:
На AlmostSorted массивах:
•	Гибридные версии быстрее стандартной на 20-40%
•	Threshold=10 показывает лучшее ускорение
На ReverseSorted массивах:
•	Умеренное ускорение 10-20% у гибридных версий
•	Threshold=50 показывает более стабильные результаты
На Random массивах:
•	Минимальная разница между алгоритмами (±5%)
•	Стандартная версия иногда немного быстрее
5. Практические  выводы:
Для каких данных выбирать:
•	AlmostSorted данные: Hybrid с Threshold=10
•	Случайные данные: Standard Merge Sort или Hybrid с Threshold=50
•	Смешанные данные: Hybrid с Threshold=50
•	ReverseSorted данные: Hybrid с Threshold=50
Оптимальный Threshold:
•	Для общего использования: 30-50
•	Для сильно частично упорядоченных данных: 10-20
•	Для полностью случайных данных: 50 (или чистый Merge Sort)
6. Теоретическое обоснование:
•	Гибридный подход сочетает O(n²) сложность Insertion Sort (эффективную на маленьких массивах) с O(n log n) сложностью Merge Sort
•	На маленьких подмассивах константы Insertion Sort лучше, чем у Merge Sort
•	На почти отсортированных данных Insertion Sort приближается к O(n)
Итоговый вывод:
Гибридный Merge Sort с правильно подобранным порогом (30-50) является оптимальным выбором для большинства практических задач, обеспечивая ускорение на частично упорядоченных данных без значительного проигрыша на случайных данных.
